Solace Agent Mesh Architecture

Dark Shift simulates a full Solace Agent Mesh — the event-driven orchestration layer that lets autonomous agents collaborate to run a manufacturing plant with minimal human intervention. Here's how every component maps to the simulation.

Core Components

Event Backbone

Pub/sub messaging layer. Every chaos event, sensor reading, and agent response flows as a topic message.

Orchestrator

Receives chaos events, decomposes them, and fans out to the right domain agents in parallel.

Domain Agents

Supply, Planner, Quality, Maintenance, Logistics — each proposes structured actions with impact scores.

Human Gateway

The player's interface. When agents can't agree or stakes are high, the human GM makes the call.

Event Flow

1. Chaos Event Published

A disruption occurs — supplier delay, equipment failure, demand surge. The event publishes to factory/chaos/{type} on the event backbone.

2. Orchestrator Subscribes

The orchestrator subscribes to factory/chaos/*. On receipt, it identifies which domain agents are relevant and fans out requests to factory/agent/{name}/request.

3. Agents Respond

Each agent evaluates the chaos in context of its domain, current factory state, and the active priority mode (Speed/Margin/Quality/Green). They publish structured responses to factory/agent/{name}/response: 

{
  "agent": "supply",
  "action": "Reroute to alt supplier",
  "impact": {
    "cost": +2000,
    "delay": -1,
    "output": 0
  },
  "confidence": 0.85
}

4. Human Gateway Decision

The player sees all proposals with trade-offs and picks one — or approves all. This decision publishes to factory/gateway/decision and the mesh executes the chosen action.

5. Factory Adapts

Stations update their throughput, KPIs shift, AMRs reroute. The event backbone carries telemetry updates on factory/cell/*/telemetry so every component stays in sync.

Topic Hierarchy

factory/
  chaos/{event-type}          # Chaos events (supplier-delay, defect-spike, etc.)
  agent/{name}/request        # Orchestrator → Agent
  agent/{name}/response       # Agent → Orchestrator
  gateway/decision            # Human → Mesh
  cell/{station}/telemetry    # Station health & throughput
  supply/eta                  # Supplier ETA updates
  kpi/{metric}                # Real-time KPI stream

Swappable Scenarios

The entire factory theme — stations, chaos cards, agents, KPIs, grades — lives in a single JavaScript config file (js/scenarios/bikes.js). To create a new scenario:

  • Copy js/scenarios/_template.js
  • Fill in your stations, chaos cards, agent responses, and mesh topology
  • Change one import line in game.html

The engine, renderer, and UI are completely scenario-agnostic. The same game loop runs a bike factory, a semiconductor fab, or a pharmaceutical plant.
