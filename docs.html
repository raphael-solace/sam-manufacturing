Solace Agent Mesh Architecture

How event-driven agent orchestration works in the simulation.

Components

Event Backbone
Pub/sub messaging layer. Every chaos event, sensor reading, and agent response flows as a topic message.
Orchestrator
Receives chaos events, decomposes them, fans out to the right domain agents in parallel.
Domain Agents
Supply, Planner, Quality, Maintenance, Logistics. Each proposes structured actions with impact scores.
Human Gateway
The player's interface. When agents can't agree or stakes are high, the human GM makes the call.

Event Flow

  1. Chaos event published to factory/chaos/{type}.
  2. Orchestrator subscribes to factory/chaos/*, identifies relevant agents, fans out requests.
  3. Agents respond with structured JSON: 
    {
  "agent": "supply",
  "action": "Reroute to alt supplier",
  "impact": { "cost": +2000, "delay": -1, "output": 0 },
  "confidence": 0.85
}
  4. Human decides — pick one proposal, approve all, or dismiss.
  5. Factory adapts — stations update, KPIs shift, telemetry flows on factory/cell/*/telemetry.

Topic Hierarchy

factory/
  chaos/{event-type}          # Disruptions
  agent/{name}/request        # Orchestrator to Agent
  agent/{name}/response       # Agent to Orchestrator
  gateway/decision            # Human to Mesh
  cell/{station}/telemetry    # Station health
  supply/eta                  # Supplier updates
  kpi/{metric}                # Real-time KPI stream

Swapping Scenarios

The entire factory theme lives in one config file. To create a new scenario:

  1. Copy js/scenarios/_template.js
  2. Fill in stations, chaos cards, agent responses, mesh topology
  3. Change one import in game.html: 
    import scenario from './js/scenarios/your-scenario.js';

The engine, renderer, and UI are scenario-agnostic.
